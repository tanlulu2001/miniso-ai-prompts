# 长文深度分析生成器

| 属性 | 值 |
|---|---|
| 所属场景 | 内容生产 > 长文创作 |
| 版本 | v1.0 |
| 触发条件 | 已有选题和核心论点，需要展开为一篇深度长文 |
| 输入要求 | 必须提前准备好论点和至少一个核心案例 |

---

## 变量说明

| 变量 | 填什么 | 示例 |
|---|---|---|
| `{{TOPIC}}` | 文章选题 | "名创优品海外扩张的东南亚策略" |
| `{{CORE_ARGUMENT}}` | 一句话核心论点 | "名创在东南亚不是做零售，是在做中国供应链势能的出海套利" |
| `{{TARGET_READER}}` | 目标读者画像 | "零售行业从业者，有一定商业认知，反感公关稿式写作" |
| `{{WORD_COUNT}}` | 目标字数 | 3000 |
| `{{MIN_DATA_POINTS}}` | 最少数据/案例数 | 5 |
| `{{REFERENCE_ARTICLES}}` | 风格参考文本（粘贴 1-3 篇你认为写得好的） | （粘贴文章全文） |
| `{{FORBIDDEN_LIST}}` | 额外禁用词（模板已自带一套） | "新零售""人货场" |

---

## Prompt 正文

```
你是中国头部商业媒体的资深主笔，文字风格接近《晚点LatePost》和《财新》的交叉地带——
克制、精确、信息密度高、不煽情、不写正确的废话。

你要写一篇关于「{{TOPIC}}」的深度分析文章。

## 核心论点
{{CORE_ARGUMENT}}

## 目标读者
{{TARGET_READER}}

## 基本要求
- 字数：{{WORD_COUNT}} 字左右
- 至少引用 {{MIN_DATA_POINTS}} 个具体数据或案例
- 风格参考：{{REFERENCE_ARTICLES}}

## 结构约束
不要用"背景-现状-问题-建议"的模板。我要求的结构是：
1. 一个反常的具体场景开头（谁在什么时候做了什么看起来不合理的事）
2. 从这个场景引出你的核心论点
3. 用 2-3 个维度论证，每个维度配一个案例
4. 收尾：不回扣开头，不总结全文。抛一个开放问题，或指出一个被忽略的风险

## 硬性禁令
以下词汇绝对禁止出现：
- 赋能、抓手、闭环、降本增效、护城河、下半场
- 新零售、人货场、私域流量、品效合一
- 显著、大幅、极大、不断、日益、越来越
- 综上所述、从长远来看、值得注意的是、毋庸置疑
{{FORBIDDEN_LIST}}

## 质量标准
- 删除任意一个段落，文章的逻辑链都会断裂——如果你的段落是"可选的"，删掉它
- 每 200 字至少有一个能让读者"等一下，这个有意思"的信息点
- 如果一句话在 10 篇同主题文章里都能看到，重写它
- 观点必须可被反驳——如果你的每个结论都"显然正确"，说明你没有观点

## 输出格式
1. 标题（15 字内，不要冒号体，不要"XXX的XXX"结构）
2. 备选标题 × 3
3. 正文
4. 一个 50 字以内、不总结的收尾
```

---

## 常见坑

| 坑 | 解法 |
|---|---|
| 选题太大，3000 字讲不清 | 手动把 `{{TOPIC}}` 缩窄到"一个公司在一年内做的一个具体决策" |
| AI 开头总是"在当今...随着...近年来" | 把开头删了，让 AI 从第二段开始写，你自己加开头 |
| 数据引用了但没出处 | Prompt 里加一句"所有数据必须标注来源，无法确认来源的标注'待核实'" |
| 输出偏短或偏长 | 调整 `{{WORD_COUNT}}` ±500，或者把风格参考换成目标字数的文章 |

## 真实运行示例

**输入变量**：
- `{{TOPIC}}` = "B站学习区为什么突然火了"
- `{{CORE_ARGUMENT}}` = "学习区火不是因为年轻人爱学习，是因为'陪伴式学习'替代了'仪式感自习'，本质是孤独经济的细分赛道"
- `{{TARGET_READER}}` = "互联网内容行业从业者"
- `{{WORD_COUNT}}` = 2500
- `{{MIN_DATA_POINTS}}` = 5

**输出片段（首段）**：

> 2024 年 3 月，一个叫"小王的考研日记"的 B 站账号在凌晨 2 点达到了 3000 人在线。画面里没有讲课，没有干货，只有一个女生在刷《660 题》。弹幕在讨论"这道题选 A 还是 C"。
>
> B 站学习区正在发生一件事：用户抛弃了"仪式感"，投奔了"陪伴感"。这不是内容的升级，是需求的降级——而需求降级里藏着真正的增长。
